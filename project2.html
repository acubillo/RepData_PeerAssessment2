NOAA Storm Data Exploration For The Most Harmful Events To Population Health And The Events That Have The Greatest Economic Consequences To The US

Adalberto Cubillo

8/22/2019

Synopsis

This report explores the U.S. National Oceanic and Atmospheric Administrations (NOAA) storm database. This database tracks characteristics of major storms and weather events in the United States, including when and where they occur, as well as estimates of any fatalities, injuries, and property damage.

The events in the database start in the year 1950 and end in November 2011. In the earlier years of the database there are generally fewer events recorded, most likely due to a lack of good records. More recent years should be considered more complete.

This data analysis main goal is to answer the following questions:

  • Across the United States, which types of events are most harmful with respect to population health?

  • Across the United States, which types of events have the greatest economic consequences?

To achieve this goal the first step was to process the storm data from NOAA database (which was provided as a comma separated values file).

Part of the data process was to perform a exploratory data analysis and transform some data (e.g.the economic damage information needed to be standardized).

After that, a comparison and a descriptive analysis was performed to establish which are the top 5 most harmful events in the US from people health and economic perspectives.

Finally, we concluded the most harmful events in the US are tornados and floods.

So, the exponent variables seems to have several values that can modify the damage data accordingly. For example, a PROPDMGEXP = M would infer the values are in millions or K will be in thousands. So we require to apply some transformations to PROPDMG and CROPDMG.

# Get the real value applying the exponent. 
apply_exp <- function(x, exp) {
  exponents <- get_exp(exp)
  result <- x * exponents
  
  return(result)
}

# Get the exponent value from the character format. 
get_exp <- function(x) {
  sapply(x, function(x) {
    if (x == "k" | x == "K") {
      result <- 1000
    } else if (x == "m" | x == "M") {
      result <- 1000000
    } else if (x == "b" | x == "B") {
      result <- 1000000000
    } else if (as.integer(x)) {
      result <- 10
    } else {
      result <- 0
    }
    
    return(result)
  })
}

# A new dataset is created only with the damage data stored. 
storm_data_dmg <- storm_data %>% 
  select(EVTYPE, PROPDMG, PROPDMGEXP, CROPDMG, CROPDMGEXP) %>% 
  mutate(prop_dmg_transf = apply_exp(PROPDMG, PROPDMGEXP),
         crop_dmg_transf = apply_exp(CROPDMG, CROPDMGEXP),
         total_dmg_transf = prop_dmg_transf + crop_dmg_transf)

Results

Now lets figure out which are the types of events that are most harmful with respect to population health in the US.

# List the top 10 elements grouped by event type 
# and based on the total population affectation (deaths + injuries). 
storm_data_population_harm <- storm_data %>% 
  select(EVTYPE, FATALITIES, INJURIES) %>% 
  group_by(EVTYPE) %>% 
  summarise(total_fatalities = sum(FATALITIES), 
            total_injuries = sum(INJURIES), 
            total_affected = total_fatalities + total_injuries) %>% 
  arrange(desc(total_affected)) %>% 
  head(10)

storm_data_economic_harm <- storm_data_dmg %>%

Further Work

As I deepen my exploratory analysis regarding the U.S. NOAA storm database, I found the following elements for improvement in the data exploration:

  • EVTYPE needs to be cleaned up and some data needs to be merged. For example, there are several categories regarding wind issues: " WIND,WND,WINDS" and Wind. Same goes for other categories.

  • PROPDMGEXP and CROPDMGEXP are not standardized across the years recorded. So, the records need to be cleaned up and reformatted. This URL present a guide to follow and understand the exponent values https://github.com/flyingdisc/RepData_PeerAssessment2/blob/master/how-to-handle-PROPDMGEXP.md
